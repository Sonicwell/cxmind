# Security Policy

## Supported Versions

CXMind currently provides security updates for the following actively maintained versions:

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take security issues seriously. If you discover a vulnerability in CXMind, please **DO NOT** create a public GitHub issue. 

Instead, report it privately to our security team:
**Email:** [security@cxmi.ai](mailto:security@cxmi.ai)

### What to include
Please provide as much information as possible to help us investigate:
* A description of the vulnerability and its impact.
* Steps to reproduce the issue.
* Any proof-of-concept (PoC) code or scripts.
* Your contact information.

### Our Response
We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress. If the vulnerability is accepted, we will coordinate the public disclosure with you after a patch has been released.
